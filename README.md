# Data Forensics Course Project- AI and Scam Detection
This prototype was conceived and developed by Neil Patel, Todd Guden, Surnilla, Vaishnavi, and Taifuk Ip. It uses logistic regression to detect scam emails, human-written text, and AI-generated text. Due to a lack of data, we cannot guarantee the effectiveness and reliability of this project with 100% certainty. The results demonstrated by this project so far are not entirely convincing for the following reasons.   
AI vs. Human Text Ambiguity—Does Logical Coherence in Sentences Equate to AI? Is AI mimicking humans, or are humans simply using correct grammar and vocabulary?​ Scam Messages Are Too Short—Most scam messages are emails or messages under 300 characters. 
It is difficult to effectively identify them within such a limited character count.​ Is Feature Selection Reliable?—Uncertain. Currently, features such as sentence length, word length, repetition, and entropy cannot serve as primary indicators to distinguish whether text is AI-generated.
Lack of data—There is not much publicly available data, and attempts to have AI generate scam messages usually fail, making it difficult to obtain data.  Due to these circumstances, we cannot guarantee the reliability of the current prototype. We hope future research will address these issues.
